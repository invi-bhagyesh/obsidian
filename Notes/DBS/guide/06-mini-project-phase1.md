# Lab 6: Mini Project (Phase I)

## Overview

This lab is about forming a team and submitting a **synopsis** for your mini project. No SQL questions to solve.

## Synopsis Format
1. **Title** and Team Members
2. **Abstract** (with modifications if any)
3. **Problem Statement** — covering both data and functional requirements

## Deliverables
1. Design and implement the database
2. Implement basic and complex queries

## Reference
Project suggestions: Chapter 9, *Database System Concepts*, Korth, 6th Edition.

## Tips
- Pick a domain you understand (library, hospital, airline, e-commerce, etc.)
- Keep ER diagram clean — 4-6 entities is enough
- Define at least 2-3 integrity constraints per table
- Plan queries of varying complexity (basic SELECT, JOINs, aggregates, subqueries)
